# Accessibility guidelines

Seabuckthorn treats accessibility as a component-boundary concern. Interactive UI is tested in Storybook with axe; site-level patterns live in Astro layouts.

## Story requirements

Every interactive component under `src/components/ui/react/` must include:

1. **Default story** — typical usage
2. **States story** — disabled, loading, error where applicable
3. **Keyboard story** — `play()` tests for non-trivial behavior
4. **A11y meta** — `parameters.a11y.test: 'error'` (set globally in [`.storybook/preview.ts`](../.storybook/preview.ts))
5. **Theme coverage** — `Light`, `Dark`, and `HighContrast` story exports for token-sensitive UI

Static `.astro` components that ship user-facing UI follow the same rules where practical. Astro composition stories (`Callout`, `SiteHeader`) use the `!test` tag and are verified manually in Storybook until Vitest loads the full Astro i18n config reliably.

## Running tests

```bash
pnpm exec playwright install chromium   # first run only (also runs on postinstall)
pnpm test:storybook
```

Failures report axe violations and failed `play()` interactions. Fix the component or story before merging.

## Theme variants

Contrast rules depend on the active theme. Interactive stories export theme-specific variants that set `globals.theme` via helpers in [`src/storybook/themeVariants.ts`](../src/storybook/themeVariants.ts).

Use the Storybook toolbar to preview themes manually.

## Overrides

If axe reports a false positive, override per story with justification:

```ts
parameters: {
  a11y: {
    // TODO(#123): third-party embed lacks accessible name
    test: "todo",
  },
},
```

Do not disable contrast checks globally.
